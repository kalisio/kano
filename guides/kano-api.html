# Using the Kano API

# Description

Kano exposes an API endpoint that can be used to query data stored in it's underlying MongoDB (opens new window) database. The API will return JSON (opens new window) documents when queried. The endpoint is located on $KANO_ROOT_URL/api/

# What's available

This depends on which layers are declared in your Kano configuration. For each layer declaring a set of features services (opens new window), the api will expose the service's data through an URL formed by the API endpoint slash the service name.

Example

If your Kano instance is exposed through https://kano.foo.xyz then the api is available on https://kano.foo.xyz/api. If for example you have the hubeau layer enabled (which instanciate both a hubeau-stations and a hubeau-observations features services), then the api will expose it's data on https://kano.foo.xyz/api/hubeau-stations and https://kano.foo.xyz/api/hubeau-observations

# Authentification

In order to access the API, clients must authenticate themselves. We use JWT (opens new window) tokens to authenticate access to Kano. Those can be added as an HTTP query parameter, using jwt=$your_token or can be passed through the Authorization HTTP header, using the Bearer scheme (opens new window).

# Queries

Queries to the Kano API are made to specific features services data which in turn query their specific underlying MongoDB collection. The way to query those collections is to use Feathers queries through REST URLs (opens new window). In addition to those, we added in the KDK (opens new window) a few handy shortcuts (opens new window) to express commonly used queries.

# Use case example

We used the Kano API to expose French nuclear power production data to a set of Grafana (opens new window) dashboards.

We first developed a Krawler (opens new window) job whose task is to scrap power production data and to push it into Kano's backing MongoDB database. On Kano's side, we added a layer declaring a feature service pointing on the database collections the Krawler job was populating.
