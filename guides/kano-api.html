# Using the Kano API

# Description

Kano exposes an API endpoint that can be used to query data stored in it's underlying MongoDB (opens new window) database. The API will return JSON (opens new window) documents when queried. The endpoint is located on $KANO_ROOT_URL/api/

# What's available

This depends on which layers are declared in your Kano configuration. For each layer declaring a set of features services (opens new window), the api will expose the service's data through an URL formed by the API endpoint slash the service name.

Example

If your Kano instance is exposed through https://kano.foo.xyz then the api is available on https://kano.foo.xyz/api. If for example you have the hubeau hydro layer enabled (which instanciate both a hubeau-hydro-stations and a hubeau-hydro-observations features services), then the api will expose it's data on https://kano.foo.xyz/api/hubeau-hydro-stations and https://kano.foo.xyz/api/hubeau-hydro-observations

# Authentication

In order to access the API, clients must authenticate themselves. We use JWT (opens new window) tokens to authenticate access to Kano. Those can be added as an HTTP query parameter, using jwt=$your_token or can be passed through the Authorization HTTP header, using the Bearer scheme (opens new window).

# Generate a JWT token

Kano supports stateless tokens, i.e. tokens not associated to a user, either to access the backend API or to automatically login when using it through an iframe. To generate a token you can use https://jwt.io with your app secret and the default header:

{
  "alg": "HS256",
  "typ": "JWT"
}

Your payload should at least look like this:

{
  "name": "your token id",
  "iss": "kalisio",
  "aud": "your app domain"
}

You can also add others token properties like an expiration date (exp).

TIP

In development mode aud=kalisio by default, all token options can be configured at authentication.jwtOptions in the API configuration.

# Queries

Queries to the Kano API are made to specific features services data which in turn query their specific underlying MongoDB collection. The way to query those collections is to use Feathers queries through REST URLs (opens new window). In addition to those, we added in the KDK (opens new window) a few handy shortcuts (opens new window) to express commonly used queries. Here are some of the most useful query examples.

# Request available stations

Typical use case is to display the stations on a map.

HTTP - GET https://your.kano.domain/api/hubeau-hydro-stations?south=44.96479793033104&north=45.02695045318546&west=-0.17578125&east=-0.087890625&jwt=your_token

# Request available observations

Typical use case is to retrieve the raw observations. You can filter spatially (i.e. bounding box) and temporally (i.e. time range) with most recent first.

HTTP - GET https://your.kano.domain/api/hubeau-hydro-observations?south=44&north=45&west=-0.5&east=0.5&time[$gte]=start_time&time[$lte]=end_time&$sort[time]=-1&jwt=your_token

# Request latest observations at (a) station(s)

Typical use case is to retrieve most recent observations for the station, or display the observations on a map if not filtering by a station ID.

HTTP - GET https://your.kano.domain/api/hubeau-hydro-observations?$groupBy=code_station&$aggregate[0]=H&$aggregate[1]=Q&properties.code_station=%23K447001001&time[$lte]=end_time&$limit=1&$sort[time]=-1&jwt=your_token

# Use case example

We used the Kano API to expose French nuclear power production data to a set of Grafana (opens new window) dashboards.

We first developed a Krawler (opens new window) job whose task is to scrap power production data and to push it into Kano's backing MongoDB database. On Kano's side, we added a layer declaring a feature service pointing on the database collections the Krawler job was populating. From that time, nuclear power production data was available for display in Kano. Here's what it looks like :

Once the collections started being populated, we looked for ways to connect Grafana to Kano's API endpoint. For this we used the Infinity Grafana datasource plugin (opens new window) allowing us to use it's JSON scrapping capabilities to feed Grafana.
